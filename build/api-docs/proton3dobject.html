Proton3DObject¶

Note

Here are the possible options for the extras parameter of the constructor:

{
    type: "cube"|"sphere"|"perspectivecamera"|"spotlight"|"sky"|undefined/null (used when importing meshes),
    fov: [A number around 1, only for cameras.],
    position: new Proton.Vector3( [Number], [Number], [Number] ),
    rotation: new Proton.Vector3( [Number], [Number], [Number] ) (Measures in degrees.),
    sun: [A Proton3D light which is the "sun" of your sky. Only for skies.],
    width: [A number. Only for cubes.],
    height: [A number. Only for cubes.],
    depth: [A number. Only for cubes.],
    radius: [A number. Only for spheres.]
    castShadow: [A boolean. Only for meshes.]
    invisible: [A boolean. Only for meshes. (Calls object.makeInvisible().)],
    noPhysics: [A boolean. Only for meshes.],
    mass: [A number. Only for meshes.],
    restitution: [A number, representing bounciness. Only for meshes.],
    friction: [A number. Only for meshes.]
}
class Proton3DObject(extras)¶

Description.

The constructor for making an object in Proton uses a variable called “extras” to prevent multiple function parameters (that must be in exact order when called because JavaScript).

Arguments

  • extras (object) – Parameters for making the object.

Proton3DObject.castShadow¶

Wether or not the object casts a shadow.

Proton3DObject.children¶

Any sub-objects of the object

Proton3DObject.mass¶

The mass of the object (0 = stationary)

Proton3DObject.onNear¶

A function to be executed when the player is within the pickup distance.

Proton3DObject.onUse¶

A function to be executed when the player is withing the pickup distance of the object, hovers over it with their crosshair, and presses their use key.

Proton3DObject.pickupDistance¶

The distance the player has to be from the object to pick it up (or whatever they’d do) by pressing their use key (typically E).

Proton3DObject.position¶

The position of the object.

Proton3DObject.receiveShadow¶

Wether or not the object can receive a shadow cast upon it. This should also work for self-shadows.

Proton3DObject.rotation¶

The rotation of the object, measured in degrees.

Proton3DObject.add(object)¶

Adds another object to the object

Arguments

  • object (object) – The Proton3DObject that will become a “child” of the main object.

Proton3DObject.addAngularVelocity(x, y, z)¶

Adds an angular velocity to the object.

Arguments

  • x (number) – The x value of the added angular velocity

  • y (number) – The y value of the added angular velocity

  • z (number) – The z value of the added angular velocity

Proton3DObject.addLinearVelocity(x, y, z)¶

Adds a linear velocity to the object.

Arguments

  • x (number) – The x value of the added linear velocity

  • y (number) – The y value of the added linear velocity

  • z (number) – The z value of the added linear velocity

Proton3DObject.animatePosition(x, y, z)¶

Eases the object into a set position.

Arguments

  • x (number) – The x value of the object’s new position

  • y (number) – The y value of the object’s new position

  • z (number) – The z value of the object’s new position

Proton3DObject.applyImpulse(force, offset)¶

Applies an “impulse” to an object.

Arguments

  • force (object) – A Proton.Vector3 object

  • offset (object) – (Optional:) A Proton.Vector3 object describing where the impulse is to be done.

Proton3DObject.changeFOV(value)¶

Changes the field-of-view for a camera.

Arguments

  • value (number) – The FOV, somewhere around 1.

Proton3DObject.delete()¶

Deletes an object. It can still be accessed as a variable, but it is deleted from the scene.

object – A Proton.Vector3 object.

Returns

function – The function itself.

Proton3DObject.getOnUse()¶

Returns the function to be called when the object is used (see the onUse property’s description for more)

Returns

function – The function itself.

Proton3DObject.getPickup()¶

Returns the variables described in the setPickup function.

Returns

object – { pickupable: [Whether the object can be picked up or not.], returnAfterPickup: [Wether the object should actually be picked up or just have the onUse function executed.] }

Proton3DObject.getPickupDistance()¶

object – A Proton.Vector3 object

  • y (number) – The y value of the vector.

  • z (number) – The z value of the vector.

Proton3DObject.makeInvisible()¶

Makes the object invisible.

Proton3DObject.makePlayer(extras)¶

Makes an object into the player. Disclaimer: Needs the imported object + this function to have the parameter noShadows set to true

Arguments

  • extras (object) – An example (all properties are optional): { type: “firstperson”|”thirdperson”, head: Proton.Vector3, invisible: boolean, noShadows: boolean, movementSpeed: number, jumpHeight: number }

Proton3DObject.remove(object)¶

Removes another object from the object

Arguments

  • object (object) – The Proton3DObject that will stop being “child” of the main object. Note that it already has to be a child of the object: you can use [object].children to see if the main object has children at all.
